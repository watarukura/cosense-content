# ToDoアプリを考える
[https://gyazo.com/7d1126b77336635e82c972c2191eb864](https://gyazo.com/7d1126b77336635e82c972c2191eb864)

銀の弾丸本(オブジェクト指向UIデザイン)を読んでOOUIに興味が出たので、ついでに自分で使いたいToDoアプリを考えてみる
(がんばればNotionでもscrapboxでもイケる気もする)

  - タスクの洗い出し やりたいことを並べてみる
  - ToDo
  - ToDoを登録する
  - ToDoを書き直す
  - ToDoを消す
  - ToDoを完了させる
  - ToDoを開始する
    - 開始と終了の時刻の差分がかかった時間
    - 休憩とかもあるから目安だけど、ないよりマシ
    - ポモドーロテクニックとの紐付けにして、nポモドーロっていう表記だとなおよし
      - 時間だと生臭いし厳密過ぎる気がする
    - 未完了のToDoを一覧する
  - ページ
    - 日付ごとに1ページ、ToDoを書ける場所
    - 前日の未完了のToDoを引き継ぐことができる
      - 引き継がなくても良い
    - 未来日のページにもToDoを書ける
      - Googleカレンダーからインポートできてもいいかも
  - タグ
    - ToDoにタグをつけられる
    - タグごとに時間を集計できる
      - タスクとかプロジェクトとかでもいいんだけど
      - ToDoの塊
      - 1つしか付けられない場合と2つ以上つけられる場合で分ける？
